## Reporting a Vulnerability

Security is a top priority for Saint-Pixels. If you believe you have discovered a vulnerability, please report it to us so we can resolve the issue as quickly as possible.

### How to Report

Please email security reports to **riviereau@outlook.com**. We request that you **do not** open public GitHub issues for security-sensitive bugs, as this alerts potential attackers to the vulnerability before a patch is deployed.

### What to include in your report:

To help us investigate, please include:

* **A clear description of the vulnerability:** What is the issue?
* **Steps to reproduce:** How can we trigger the vulnerability?
* **Proof of Concept (PoC):** If possible, provide a small script or steps showing the exploit.
* **Environment details:** Which browser or OS were you using?

### What to expect:

1. **Acknowledgement:** You will receive a response within **48–72 hours** confirming receipt of your report.
2. **Investigation:** We will evaluate the report to determine if it is a valid security risk.
3. **Patching:** If validated, we will work on a fix. We will keep you updated on the progress and expected timeline for the fix.
4. **Disclosure:** We ask that you keep the report confidential until a fix is released. Once live, we will be happy to credit you for the discovery in our release notes if you wish.

## Community Security & Conduct

While security vulnerabilities involve code, the "social" security of our canvas is equally important. To keep Saint-Pixels a fun and safe space, we have the following rules for collaborative conduct:

### Art Respect Policy

* **Respect others' art:** Do not maliciously erase or overwrite another user's drawing.
* **No cheating:** No botting, automation scripts, or unfair third-party tools.
* **Avoid "Griefing":** Do not excessively bleach, erase, or overwrite another user's work for no valid reason. Collaboration and fixing stray pixels are encouraged, but destructive behavior is not.

### Enforcement

Users who consistently break these rules or attempt to bypass IP-based rate limits/cooldowns may have their access restricted. We use logging to track pixel placement patterns to help maintain a healthy canvas environment.

### Code of Conduct

Please act in good faith. Do not use vulnerabilities to harm the community, extract user data, or maliciously interfere with the service. Researchers who report issues ethically are valued members of our community.
